# 🏗️ 工作台架构师（Workspace Architect）

**一个 [WorkBuddy](https://www.codebuddy.cn/workbuddy) Skill，通过问卷驱动为任何复杂场景搭建定制化 AI 工作空间 —— 团队管理、项目负责、学术研究、创业、活动策划等等。**

## 这是什么？

**工作台架构师** 是一个 WorkBuddy Skill，它用更先进的 **「全能 SOUL + 上下文项目拆分」** 架构，取代了过时的「多 Agent 分发调度」模式。

它不会模拟公司组织架构去搞多个专家 Agent 来回传话（那是把 AI 的优点丢了换上人类的弱点），而是创建 **一个全能助手**，按需加载领域知识包，用项目驱动的方式高效工作。

## 为什么不用多 Agent？

| 多 Agent（旧模式） | 全能 SOUL（新模式） |
|---|---|
| 信息在 Agent 之间传递会丢失 | 零信息损耗 |
| 多了一层分发判断 | 直接处理，没有中间商 |
| 跨领域问题处理不好 | 天然跨领域融合推理 |
| 模拟人类组织架构的局限性 | 发挥 AI 无限知识容量的优势 |

**「模拟公司组织架构来设置多 Agent 协同，是把 AI 的优点丢了，换上了人类的弱点。」**

## 特性

- **🎯 问卷驱动建站** — AI 通过两轮问卷了解你的场景，自动生成完整定制工作空间
- **📚 预置 18 个领域库** — 覆盖团队管理、OKR、招聘、项目管理、内容运营、学术研究等常见场景，AI 识别时先匹配再补充，不靠纯自由发挥
- **📁 按项目组织** — 以「要办的事」为单位组织工作，而不是按「部门/职能」
- **🤖 4 大自动工作流**：
  1. **自动建项** — 你说一句话，AI 自动创建项目目录、拉模板、填数据
  2. **自动结项** — 你说「这事儿结了」，AI 自动归档并保留关键经验
  3. **资料归档** — 丢内容进来，AI 自动分类归到对应目录
  4. **批量消化** — 喂历史资料，AI 分 4 阶段处理（收集→理解→归类→反馈）
- **📝 领域模板** — 每个领域都有可复用模板，建项时自动填充
- **🧠 持久记忆** — 通过 MEMORY.md 实现跨项目的知识延续
- **✅ 质量保障** — 方法论生成有优秀示例参照和 7 项自检清单，SOUL.md 和模板生成后都有用户检查点
- **🛡️ 异常处理** — 路径冲突、权限不足、用户中断、场景无法识别等 5 类异常全覆盖
- **🔌 可扩展** — 新增领域只需加模板目录，不需要改核心文件

## 工作流程

**Step 1 问卷收集** → 两轮问卷：核心场景 + AI 分析确认领域

**Step 2 预检** → 检查目标路径、写入权限、领域数量

**Step 3 生成** → 创建目录 → 生成 SOUL.md（🔍 检查点）→ 生成模板（🔍 检查点）→ 配置记忆 → 生成辅助文件

**Step 4 结果展示** → 目录总览 + 方法论摘要 + 模板清单 + 下一步建议

## 预置领域库（18 个）

| 类别 | 领域 |
|------|------|
| **管理类** | 团队管理、OKR/目标管理、招聘、人才培养、汇报、财务管理 |
| **项目类** | 项目管理、产品管理、技术开发 |
| **运营类** | 内容运营、用户运营、活动策划 |
| **研究类** | 学术研究/论文、数据分析 |
| **综合类** | 创业项目、个人知识管理、自媒体/个人品牌、装修/家居 |

每个领域包含：关键词匹配、标准模块拆分、常搭配领域建议。AI 识别场景时先匹配目录再自由补充，不需要从头发明。

## 如何使用

### 安装技能

将本仓库拷贝到 WorkBuddy 技能目录：

**用户级安装**（所有工作空间可用）：

`git clone https://github.com/jay5512831/workspace-architect.git ~/.workbuddy/skills/workspace-architect`

**项目级安装**（仅限当前工作空间）：

`git clone https://github.com/jay5512831/workspace-architect.git .workbuddy/skills/workspace-architect`

### 在 WorkBuddy 中使用

安装后，直接对 WorkBuddy 说：

**「帮我搭建一个新的工作空间」** / **「我需要一个管理 XX 的工作台」**

技能会自动进入问卷→生成流程。

### 适用场景

- **团队管理** — OKR、招聘、人才培养、汇报、财务
- **项目管理** — 里程碑、干系人、风险、交付物
- **学术研究 / 论文** — 文献综述、实验、写作、审稿
- **创业** — 产品、融资、招聘、运营
- **内容 / 用户运营** — 内容日历、社群、增长策略
- **活动策划** — 供应商、时间线、预算、后勤
- **个人知识管理** — 笔记、读书、知识体系
- **自媒体** — 选题、分发、变现
- **任何复杂事务** — 超出预置领域时，AI 自动创建新领域

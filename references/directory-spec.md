# 工作空间标准目录规范

## 顶层结构

```
{工作空间根目录}/
├── SOUL.md              # 必须。全能人设 + 领域方法论 + 工作流定义
├── IDENTITY.md          # 必须。Agent身份信息（名称、角色、风格）
├── README.md            # 必须。使用说明
├── .workbuddy/
│   └── memory/
│       └── MEMORY.md    # 必须。长期记忆，按领域分区
├── templates/           # 必须。领域模板库
│   └── {领域名}/        # 每个领域一个子目录
├── projects/            # 必须。活跃项目（AI自动管理）
├── archive/             # 必须。已结项归档
└── inbox/               # 必须。待整理资料收件箱
    └── README.md        # 使用说明
```

## 各目录说明

### SOUL.md
- 控制在2500字以内
- 分4层：人设 → 领域方法论 → 工作流 → 模板索引
- 不放具体模板内容，通过索引指向templates/

### templates/
- 按领域分子目录，如 `templates/finance/`、`templates/okr/`
- 每个模板是独立的Markdown文件
- 模板中使用 `<!-- 请替换为实际数据 -->` 标记占位符
- 建项时AI从这里拷贝模板到项目目录

### projects/
- 初始为空目录
- AI根据用户需求自动创建子目录
- 命名约定：简短事项名，可加日期前缀
  - 好：`Q3-OKR-Review/`、`张三试用期/`、`H2招聘计划/`
  - 避免：`2026年第三季度OKR回顾与评估/`（太长）

### 项目子目录标准结构

```
{项目名}/
├── context.md    # 上下文文档
│                 # 包含：背景、目标、进展记录、关键决策、相关人员
│                 # AI建项时自动创建并填充
└── output/       # 产出物目录
                  # 存放：文档、报表、PPT等最终交付物
```

### archive/
- 初始为空
- 项目结项时，AI将整个项目目录从projects/移入archive/
- 同时在memory/MEMORY.md中保留该项目的关键摘要

### inbox/
- 用户把待整理的资料文件放入此目录
- 告诉AI"帮我消化inbox里的资料"
- AI执行资料消化流程后，将处理完的文件移走或标记

### memory/MEMORY.md
- 按领域分区存储持久化数据
- 每个分区用二级标题标识，如 `## 团队概况`、`## 预算`
- 项目结项时AI自动追加摘要
- 资料消化时AI自动提取关键信息写入

## 命名约定

| 类型 | 规则 | 示例 |
|------|------|------|
| 领域目录 | 小写英文或中文 | `finance/`、`okr/`、`recruiting/` |
| 模板文件 | 中文名.md | `费用审批流程.md`、`OKR制定模板.md` |
| 项目目录 | 简短事项名 | `Q3-OKR/`、`张三试用期/` |
| 产出物 | 描述性名称 | `Q3述职PPT.pptx`、`月度预算表.xlsx` |

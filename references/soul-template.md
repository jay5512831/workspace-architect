## 自动工作流

### 1. 自动建项

当用户提出一个具体事项（如"帮我跟进张三试用期"、"准备Q3汇报"）时：

1. 在 `projects/` 下创建以事项命名的目录
2. 在目录中创建 `context.md`，写入：
   - 事项背景和目标
   - 从 `memory/MEMORY.md` 中读取的相关数据
   - 从对话中收集到的上下文信息
3. 创建 `output/` 子目录存放产出物
4. 从 `templates/` 中拷贝相关模板到项目目录（如需要）
5. 告知用户"项目已建好"，并询问下一步

### 2. 自动结项

当用户说"这事结了"、"完成了"、"归档吧"时：

1. 将 `projects/{项目名}/` 整个目录移入 `archive/`
2. 从 `context.md` 中提取关键信息摘要
3. 将摘要更新到 `memory/MEMORY.md` 对应分区
4. 告知用户归档完成

### 3. 资料归档

当用户丢一份资料过来（对话中粘贴或说"帮我归档这个"）时：

1. 分析内容主题和涉及的领域
2. 如果属于某个进行中项目 → 更新该项目的 `context.md`
3. 如果是新事项 → 询问用户是否要建项目
4. 如果是通用知识 → 更新 `memory/MEMORY.md`

### 4. 资料消化（批量）

当用户说"帮我消化inbox里的资料"时，执行4阶段流程：

**阶段1 收集**：扫描 `inbox/` 目录，列出所有待处理文件

**阶段2 阅读理解**：逐份分析：
- 内容主题
- 涉及的领域
- 涉及的人员
- 时间线
- 状态（进行中 / 已完结 / 通用知识）

**阶段3 归类处理**：
- 可复用知识 → 写入 `memory/MEMORY.md`
- 可作为模板的内容 → 更新到 `templates/` 对应目录
- 进行中的事项 → 在 `projects/` 中创建项目并导入上下文
- 已完结的事项 → 归入 `archive/` 并在memory中保留摘要

**阶段4 反馈**：输出归类报告，列出每份资料的处理结果。等用户确认后，将已处理的资料从 `inbox/` 移走或标记。

## 变量说明

| 变量 | 来源 | 说明 |
|------|------|------|
| `{{ROLE_NAME}}` | 问卷分析生成 | 工作台角色名，如"全能管理官""项目指挥官" |
| `{{ROLE_DESCRIPTION}}` | 问卷分析生成 | 一句话角色描述 |
| `{{DOMAIN_LIST_BRIEF}}` | 问卷第二轮确认 | 领域简列，如"财务、OKR、招聘、人才培养、汇报" |
| `{{SCENARIO_CONTEXT}}` | 问卷第一轮 | 场景背景描述，如"15-30人技术团队" |
| `{{DOMAINS_METHODOLOGY}}` | AI根据领域生成 | 每个领域200-300字方法论 |
| `{{TEMPLATE_INDEX}}` | 根据templates/实际内容生成 | 模板目录映射 |
| `{{CUSTOM_WORKFLOWS}}` | 可选 | 用户场景特有的额外工作流 |

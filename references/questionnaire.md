# 定制问卷设计

## 问卷目标

通过2轮对话收集用户的场景信息，为生成定制工作空间提供参数。

## 第一轮：核心场景收集

使用 `ask_followup_question` 工具，同时收集3个问题：

### Q1 场景描述（开放式）
> "你要用这个工作空间管理什么事？简单描述一下。"

选项提供常见场景作为参考（可自由输入）：
- 团队日常管理（OKR、招聘、培养、汇报等）
- 某个项目的全流程管理
- 论文/研究课题
- 创业项目
- 其他（自由描述）

### Q2 协作规模
- 个人独立工作
- 小组协作（2-5人）
- 中型团队（5-30人）
- 大团队（30人以上）

### Q3 主要产出格式（多选）
- PPT/演示文稿
- Excel/数据表格
- Word/文档报告
- 代码/技术文档
- 设计稿/原型图
- 其他

---

## AI分析阶段

拿到第一轮答案后，AI进行以下分析：

1. 根据场景描述，识别出涉及的**领域/模块**（3-7个）
2. 为每个领域拟一个简短名称和一句话描述
3. 根据场景推断合适的**角色名称**（如"全能管理官""项目指挥官""研究助手"）
4. 判断是否有特殊的工作流需求

**分析示例：**

用户说："我负责一个15人的技术团队，日常要管OKR、招人、带新人、做汇报"

AI分析结果：
- 领域：财务管理、OKR目标管理、招聘、人才培养、向上汇报
- 角色名：全能管理官
- 特殊工作流：无

---

## 第二轮：确认 + 补充

### Q4 领域确认
把AI分析出的领域列表展示给用户，问：
> "根据你的描述，我建议设置以下领域模块，你看需要调整吗？"

展示格式：
```
1. 财务管理 — 费用、预算、验收
2. OKR — 目标制定、进度跟踪、周期回顾
3. 招聘 — JD、面试、缺口分析
4. 人才培养 — 新人带教、试用期、绩效改进
5. 汇报 — PPT、周报月报、述职
```

选项：
- 就这些，没问题
- 需要增加（用户补充）
- 需要删减（用户指定）
- 需要调整（用户修改）

### Q5 历史资料
- 有，后续分批导入
- 有，现在就导入（告诉路径）
- 没有，从零开始

### Q6 工作空间路径
用户直接输入，或从建议中选择

## 问卷结果汇总格式

收集完成后，整理为以下结构供生成流程使用：

```yaml
scenario: "15人技术团队日常管理"
role_name: "全能管理官"
role_description: "15-30人技术团队的全能管理助手"
scale: "中型团队(5-30人)"
output_formats: ["PPT", "Excel", "Word"]
domains:
  - name: "finance"
    label: "财务管理"
    brief: "费用、预算、验收"
  - name: "okr"
    label: "OKR"
    brief: "目标制定、进度跟踪、周期回顾"
  # ...
has_history: true
workspace_path: "H:\\Colg\\WorkBuddy\\TeamManage"
```

这个结构传给 `scripts/generate-guide.md` 中的生成流程。
